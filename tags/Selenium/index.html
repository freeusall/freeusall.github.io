Learn Selenium WebDriver Part I

I am learning this with Java, Firefox, Eclipse on Windows 7.

Today I started to learn Selenium Web Driver API

(Note: readers in China, this is outside the wall)

I wanted to use Selenium Java Firefox Web Driver in an Eclipse project on Windows 7.

Here I met with a couple of issues with the part “Setting Up a Selenium-WebDriver Project”.

But fortunately, I finally solved them and successfully ran the first simple example.

Set up the dev environment

Before you start, make sure that the following are properly installed:

  • Java 1.7.0_45 x64
  • Eclipse Version: Kepler Service Release 2
  • Apache Maven 3.2.1
  • m2e (Maven Integration for Eclipse) 1.5.1 (Note: readers in China, this seems to be outside the wall)

Install Maven dependency libraries

(1) Create a temp folder, 

D:\mySVN\Selenium\Temp1

I will refer to it as %TEMP1%.

(2) Under %TEMP1%, create

pom.xml

(3) Copy the pom settings in the tutorial into pom.xml.

(4) From a command-line, CD into %TEMP1% directory and run maven as follows.

mvn clean install

This will download all dependencies in your maven M2 REPO directory.

Generate an empty Eclipse project

(1) Create another temp folder

D:\mySVN\Selenium\Temp2

I will refer to it as %TEMP2%.

(2) From a command-line, CD into %TEMP2% directory and run maven as follows.

mvn eclipse:eclipse

This will make an empty Eclipse project.

Import

(1) Create the workspace dir

D:\mySVN\Selenium\WebDriverExample

I will refer to it as %EXAMPLE_BASE%

(2) Open Eclipse, and open workspace in %EXAMPLE_BASE%

(3) Import the empty project into workspace by copy files

File - Import - Existing Project - browse to %TEMP2% - make sure to check “copy file “

Then, you got a dir

D:\mySVN\Selenium\WebDriverExample\MySel20Proj

I will refer to it as %EXAMPLE_PRJ%

The Question:　How to Continue

After this, there is no instruction. So I was in dark.

Compilation was ok. But I have no clue how to continue.

Since I got stuck, I decided to search for answers.

During search, I tried two other tutorials.

Other Tutorials: Advanced Selenium

The first is an advanced real tutorial

Configure Selenium Continuous Integration with Maven

It worked perfectly. But it is a bit complicated for me now.

Other Tutorials: Maven Tutorial

I quickly went over the first chapters of

Maven: The Complete Reference

Then, I tried the most simple maven helloworld.

It also worked.

I learnt a few important relevant points:

  • Maven works around default conventions, including project dir structures and lifecyles.

  • If you follow the conventions, you don’t need to do much.

  • If you deviate, you still can do, but need much work.

Finally, I got the idea about how to configure the project.

(1) Convert the project MySel20Proj to Maven project

In Eclipse package explorer, right click “MySel20Proj” -> Configure -> convert to Maven project

(2) Create default Maven dir structure according to package info (org.openqa.selenium.example) of the example class Selenium2Example

From a command-line, CD into %EXAMPLE_PRJ% directory and run cmd as follows.

mkdir src\main\java\org\openqa\selenium\example
mkdir src\test\java\org\openqa\selenium\example

(3) Add class Selenium2Example

Then in Eclipse package explorer, under src/main/java, right click package org.openqa.selenium.example, add a new class Selenium2Example.

Then copy the code from the example into it.

(4) Add a dummy JUNIT Test

Then in Eclipse package explorer, under src/test/java, right click package org.openqa.selenium.example, add a new JUNIT (let’s say, MyTest.java).

Just leave the default implementation intact.

Make sure that in Eclipse package explorer, you see no error.

Click “Run As …” button in the toolbar.

On your first run, you need to type classname “Selenium2Example” (typing an “S” will show it) as an argument to jar command.

Now there is an exception: unable to find firefox binary in PATH env variable.

Add firefox.exe path to machine PATH env variable.

Again run the application

You should see that Firefox opens. When google shows up, “Cheese!” is typed in.

It runs and closes very quickly, before you can see the result page clearly.

But you will see messages in the console window.

Notes for readers in China

As you know, google is outside the wall. So please change the url in the example to baidu, and change element id or name accordingly.
