How did I set up this blog on GitHub using Hexo

The local environment is Windows7 x64.
The remote server is www.github.com

I. INSTALL SOFTWARE

  1. Install Git

    Go to
    http://git-scm.com/download/win

    Download the latest Windows installer.
    I got

      Git-1.9.2-preview20140411.exe

    Install it
    You could use instrctions such as this (it’s in Chinese):

    学习笔记--Git安装、图文详解
http://www.csdn123.com/html/mycsdn20140110/54/54c848785010f5382823482e407ccf76.html

    Following the official instruction is also a good idea.

    Then, you should see “GIT” folder in “Start Menu\Programs”. In this folder, you should see

    GIT Bash
    GIT GUI

    In your right click menu, you should also see

    GIT Init here
    GIT Bash
    GIT GUI

    However, in order to conveniently use Hexo later, you should also add GIT bin path to PATH system environment variable.
    After this, you should restart your machine to let it take effect.

    Make sure in cmd, when you type “git”, you get its help.

  2. Install Node.js

    Go to
    http://www.nodejs.org/download/

    Download the latest Windows msi
    I got

      node-v0.12.0-x64.msi

    Install it.

    Then, you should see “Node.js” folder in “Start Menu\Programs”. In this folder, you should see
    Node.js command prompt
    Node.js documentation.url
    Node.js website.url
    Node.js
    Uninstall Node.js

    Make sure in cmd, when you type “node”, you get prompt “>” waiting for your input.

  3. Install npm

    Make a dir for npm. Mine is
    D:\myGIT\npm

    In this folder, right click, and click “GIT Bash”.

    In GIT Bash, execute
    git clone —recursive git://github.com/isaacs/npm.git

    Click “Node.js command prompt” in “Start Menu\Programs\Node.js”

    In Node.js command prompt window, execute
    cd D:\myGIT\npm
    node cli.js install npm -gf

    Make sure in cmd, when you type “npm”, you get its help.

    Ref:
    STEP 1 and 2 in “npm安装图文教程”

      http://jingyan.baidu.com/article/a17d528506d7f58098c8f2b0.html

    Ignore STEP 3 and 4 if you don’t need Express

  4. Install hexo

    In Node.js command prompt window, execute
    npm install -g hexo

    Make sure in cmd, when you type “npm”, you get its help.

II. CREATE BLOG

  1. Make a dir for hexo blog.

    Mine is
    D:\myGIT\hexo

  2. In this folder, right click, and click “GIT Bash”

  3. In GIT Bash, execute

    hexo init
    npm install
    hexo g
    hexo s

  4. Open localhost:4000 in a browser, you should see blog

III. GITHUB ACTIONS

  1. Sign up to GITHUB using a valid e-mail, and remember your

    github user name
    github login password

  2. Add a repository named “yourusername.github.io”.

    Please substitute yourusername with your own.

IV. CONFIG GIT

  1. In GIT Bash, execute the following using your own user name and email
    git config —global user.name “yourusername”
    git config —global user.email “youremail”

V. CONFIG HEXO DEPLOYMENT (HTTPS)

  1. open _config.yml, and configure deployment as

    deploy:
    type: github
    repo: https://github.com/yourusername/yourusername.github.io.git
    branch: master

VI. YOUR FIRST POST

  1. Go to hexo blog dir.
    Mine is
    D:\myGIT\hexo

  2. In this folder, right click, and click “GIT Bash”

  3. In GIT Bash, execute
    hexo -n “my first blog”

  4. You should see a new post in “\source_posts” in your hexo blog dir
    \hexo\source_posts\my-first-blog.md

  5. Edit the md file as you like.
    Be sure to follow hexo instructions.

  6. If the post contains international characters, such as Chinese characters, save it in “UTF8 without BOM” format.

VII. PUSH YOUR POST

  1. Go to hexo blog dir.
    Mine is 

     D:\myGIT\hexo

  2. In this folder, right click, and click “GIT Bash”

  3. In GIT Bash, execute

    hexo clean
hexo g
hexo s

    Check your post locally at localhost:4000.

  4. If ok, then In GIT Bash, execute
    hexo d

    you should see that deployment is completed

  5. If everything is ok, you should see your post in
    http://yourusername.github.io

    Replace yourusername with your own.

  6. Ref:
    hexo系列教程：（二）搭建hexo博客 (it’s in Chinese)
    http://zipperary.com/2013/05/28/hexo-guide-2/

VIII. DISCUSSIONS

  1. hexo clean should not be shortened to “hexo c”, which does not work

  2. If “hexo clean” failed, you could get multiple copies of the same post.
    In that case, you could manually remove hexo\db.json and hexo\public

  3. With this method, hexo dir should not become git repo using “git init”

  4. Hexo deployment can alternatively use SSH method, but I did not do it successfully.
