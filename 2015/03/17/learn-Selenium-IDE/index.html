learn Selenium IDE

This week I started to learn Selenium.
The reason is that it is open source, and widely used.

I decided to first go over the official tutorial.
http://docs.seleniumhq.org/docs/
(Note for readers in China: this site is outside the wall.)

Today is about Selenium-IDE, a Firefox add-on that can produce web app test cases by recording user actions.

Here is what I learned.

Selenium Commands Types

  • Action
  • Accessor
  • Assertion
    — assert (on failure stops)
    — verify (on failure contiues)
    — waitfor (on falure/timeout stops)

Selenium Command Components

  • Command
  • Target (first parameter)
  • Value (second parameter)

Locator

  • By Identifier (By Id or By Name)
  • By Id (id=)
  • By Name (name=)
  • By XPath (//)
  • By Link (link=)
  • By Dom (dom=)
  • By CSS (css=)

Matching Text Patterns

  • glob:
    *
    []

  • regexp:

  • regexpi:

  • exact:

Waiting

  • AndWait (New page or Refresh)
  • waitFor (AJAX, no page refresh, but certain element refresh)

Control Flow Methods

  • Selenium-RC
  • Javascript
  • goto_sel_ide.js

Store

  • Command
  • Variable
  • ${}

Javascript and Selenese Parameter

  • Command that takes Script Parameter
    — assertEval
    — verifyEval
    — storeEval
    — waitForEval

  • JavaScript Parameter
    — javascript {}

echo Command

  • Text
  • Variable
